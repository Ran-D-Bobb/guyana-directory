# Full SEO Audit Report: waypointgy.com

**Date:** February 21, 2026
**Business Type Detected:** Local Business Directory / Tourism Platform (Guyana)
**Pages Analyzed:** 12+ pages across all sections
**Overall SEO Health Score: 28/100**

---

## Executive Summary

waypointgy.com is a Next.js 15 Progressive Web App serving as a local business directory and tourism platform for Guyana. While the site has solid frontend technology and PWA capabilities, it has **severe SEO deficiencies** that are preventing it from being indexed by Google entirely. The site currently has **zero indexed pages** in Google search results.

### Top 5 Critical Issues
1. **No robots.txt file** - Search engines have no crawling directives
2. **No sitemap.xml** - Search engines cannot discover pages
3. **No canonical URLs** on any page - Duplicate content risk
4. **Generic/duplicate title tags** - 5 of 7 main pages share the same title
5. **Zero structured data (JSON-LD)** - No rich snippet eligibility

### Top 5 Quick Wins
1. Add `robots.txt` to `/public` directory (5 min fix)
2. Add dynamic `sitemap.xml` via Next.js route (30 min)
3. Add unique metadata to `/businesses`, `/tourism`, `/events` pages (1 hr)
4. Add canonical URLs via Next.js metadata API (30 min)
5. Add Open Graph tags to the root layout (15 min)

## 5. Performance / Core Web Vitals (Score: 6/10)

### Positive Indicators
- Next.js 15 with server-side rendering
- Image optimization via Next.js Image component
- Font display swap (no FOIT)
- Service worker with smart caching strategies
- Stale-while-revalidate for listing/detail pages

### Concerns
- Heavy JavaScript bundle (React + Next.js framework)
- Multiple external image sources (Unsplash, Supabase, ftcdn)
- Client-side rendering for dynamic content may affect LCP
- No explicit critical CSS inlining detected

### Recommendations
- Run Lighthouse audit for actual CWV numbers
- Implement `loading="lazy"` on below-fold images
- Consider static generation (SSG) for category/listing pages
- Audit third-party script impact
